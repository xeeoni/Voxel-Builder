# 복셀 빌더 (Voxel Builder)

브라우저에서 바로 실행되는 로블록스/마인크래프트 스타일의 3D 자유 건축 샌드박스 게임입니다.
Three.js만으로 만든 **순수 HTML/JS 파일 하나**라 별도 빌드 과정 없이 GitHub Pages에 올리면 누구나 링크만으로 접속해 플레이할 수 있습니다.

## 주요 기능

- **1인칭 / 3인칭 시점 전환** (`V` 키 또는 화면 우측 버튼) — 1인칭은 캐릭터 눈높이에서, 3인칭은 마우스로 자유롭게 회전하는 궤도 카메라로 플레이
- **블록 텍스처** — 모든 블록에 코드로 직접 그린 픽셀 아트 텍스처 적용 (외부 이미지 파일 없이 캔버스로 실시간 생성). 잔디는 옆면에 흙+잔디 경계, 나무는 나이테/나무결, 벽돌은 벽돌 줄눈, 유리는 창틀 무늬가 있는 등 블록마다 다른 패턴
- **낮/밤 주기** — 해가 천천히 움직이며 하늘색과 조명이 낮 → 노을 → 밤으로 자연스럽게 변화하고, 밤에는 별이 뜸
- **그림자** — 태양광 기준 실시간 그림자로 블록과 캐릭터에 입체감 추가
- **비행 모드** (`F` 키) — 중력 없이 자유롭게 날아다니며 건축 가능 (`Space` 상승 / `C` 하강), 블록과의 충돌은 그대로 유지되어 벽을 통과하지 않음
- **실행 취소** (`Ctrl+Z`) — 방금 부수거나 설치한 블록을 되돌리기
- 기본 블록형(로블록스 스타일) 캐릭터 제공, 이동 시 팔다리가 자동으로 움직이는 걷기 애니메이션 포함
- **직접 만든 3D 모델(.glb/.gltf)을 불러와 캐릭터로 사용** — 시작 화면 또는 일시정지(`Esc`) 화면에서 파일 선택 시 자동 적용, 모델에 애니메이션 클립이 있으면 걷기/대기 동작에 자동 매핑
- WASD 이동은 카메라 기준으로 동작하며 캐릭터가 이동 방향으로 자연스럽게 회전
- 점프 / 중력 / 블록과의 충돌 처리
- 좌클릭으로 블록 부수기, 우클릭으로 블록 설치 (캐릭터로부터 일정 거리 이내에서만 가능)
- 잔디·흙·돌·나무·모래·벽돌·눈·금·유리 9종 블록 (숫자키 1–9로 선택)
- 만든 세계는 브라우저의 `localStorage`에 자동 저장 → 새로고침해도 유지
- 시작 화면에서 "저장된 세계 초기화" 가능

## 로컬에서 바로 실행하기

파일을 다운로드한 뒤 `index.html`을 더블클릭해서 브라우저로 열기만 하면 됩니다.
(인터넷 연결은 Three.js 라이브러리를 CDN에서 불러오기 위해 필요합니다.)

## GitHub에 올려서 실행하는 방법 (GitHub Pages)

1. GitHub에서 새 저장소(Repository)를 만듭니다. 예: `voxel-builder`
2. 이 폴더 안의 `index.html`을 저장소 루트에 업로드(또는 `git push`)합니다.
   ```bash
   git init
   git add index.html README.md
   git commit -m "voxel builder game"
   git branch -M main
   git remote add origin https://github.com/<사용자명>/voxel-builder.git
   git push -u origin main
   ```
3. GitHub 저장소 페이지에서 **Settings → Pages**로 이동합니다.
4. **Source**를 `Deploy from a branch`로 설정하고, Branch를 `main` / `/(root)`로 선택한 뒤 저장합니다.
5. 잠시 후 `https://<사용자명>.github.io/voxel-builder/` 주소로 접속하면 게임이 바로 실행됩니다.

> 별도의 서버나 빌드 도구가 필요 없는 정적 HTML이라 GitHub Pages, Vercel, Netlify 등 아무 정적 호스팅에도 그대로 올릴 수 있습니다.

## 조작법

| 동작 | 키 |
|---|---|
| 이동 | `W` `A` `S` `D` |
| 점프 (걷기 모드) / 상승 (비행 모드) | `Space` |
| 하강 (비행 모드) | `C` |
| 달리기 | `Shift` |
| 카메라 회전 | 마우스 이동 |
| 카메라 줌 (3인칭에서만) | 마우스 휠 |
| 블록 부수기 | 좌클릭 |
| 블록 설치 | 우클릭 |
| 블록 선택 | `1`–`9` |
| 1인칭 / 3인칭 전환 | `V` |
| 비행 모드 켜기/끄기 | `F` |
| 실행 취소 | `Ctrl` + `Z` |
| 일시정지 / 캐릭터 모델 변경 | `Esc` |

## 3D 캐릭터 모델 불러오기

시작 화면 또는 게임 중 `Esc`로 일시정지한 화면에서 "3D 캐릭터 모델 불러오기" 버튼으로 `.glb` 또는 `.gltf` 파일을 선택하면 즉시 캐릭터가 교체됩니다.

- 모델은 자동으로 높이 약 1.7m로 스케일 조정되고, 발이 바닥(y=0)에 오도록 위치가 보정됩니다.
- 모델에 애니메이션 클립이 포함되어 있으면 이름에 `walk`/`run`이 들어간 클립을 걷기 동작으로, `idle`/`stand`가 들어간 클립을 대기 동작으로 자동 인식합니다. 해당 이름의 클립이 없으면 첫 번째 클립을 사용합니다.
- 파일은 서버로 업로드되지 않고 브라우저 안에서만 읽어 처리되므로 별도의 백엔드가 필요 없습니다.
- Blender, Mixamo(캐릭터+애니메이션을 glTF로 내보내기), Sketchfab 등에서 만든 `.glb` 파일이 잘 동작합니다. 다만 브라우저 세션 간에는 모델이 저장되지 않으므로, 새로고침하면 다시 불러와야 합니다.
- "기본 캐릭터로 되돌리기" 링크(일시정지 화면)로 로블록스 스타일의 기본 블록형 캐릭터로 언제든 복귀할 수 있습니다.

## 커스터마이징 팁

- `BLOCK_TYPES` 배열(`index.html` 상단 스크립트)에 항목을 추가하면 새로운 블록 종류를 쉽게 추가할 수 있습니다.
- `generateFlatWorld()` 함수를 수정하면 시작 지형(크기, 언덕, 나무 등)을 원하는 대로 바꿀 수 있습니다.
- `buildDefaultCharacter()` 함수에서 기본 캐릭터의 형태와 색상을 수정할 수 있습니다.
- `camDistance`, `EYE_HEIGHT`, `PITCH_MIN`/`PITCH_MAX` 값을 조정하면 카메라의 기본 거리와 회전 범위를 바꿀 수 있습니다.
- `draw그래픽함수`(예: `drawGrassTop`, `drawWood`, `drawBrick` 등)를 수정하면 블록 텍스처의 색상과 패턴을 원하는 대로 바꿀 수 있습니다. 캔버스 2D API로 그리기 때문에 별도 이미지 파일이 필요 없습니다.
- `DAY_LENGTH` 값(초 단위)을 조정하면 낮/밤 주기의 길이를 바꿀 수 있습니다.
- `FLY_SPEED`, `WALK_SPEED`, `SPRINT_SPEED` 값으로 이동 속도를 조정할 수 있습니다.
- 현재는 블록마다 개별 Mesh를 사용해 단순하게 구현되어 있어, 수만 개 이상의 블록을 쌓으면 프레임이 떨어질 수 있습니다. 더 큰 세계가 필요하다면 `InstancedMesh` 기반으로 리팩터링하는 것을 추천합니다.
- 여러 사람이 함께 접속하는 멀티플레이어는 포함되어 있지 않습니다. 추가하려면 WebSocket 서버(예: Node.js + `ws`, 또는 Firebase Realtime Database)와 연동해 블록 변경 이벤트를 다른 플레이어에게 브로드캐스트하면 됩니다.
